# Avoiding Common Vibe-Coded Website Mistakes

This skill captures design critique wisdom from Y Combinator's "Common Mistakes With Vibe Coded Websites" video
(https://www.youtube.com/watch?v=DNSXlBmukck), featuring YC's Aaron Epstein and Visiting Partner Raphael Schaad
(co-founder of Cron, acquired by Notion).

As AI-assisted design tools become more common, websites are increasingly exhibiting a recognizable set of "tells"
that signal the design was generated by an LLM rather than thoughtfully crafted. These tells erode trust and make
sites feel generic. This guide helps you identify and avoid them.

## The Core Anti-Patterns (Ranked by Impact)

### 1. Purple/Dark Gradients as Default Aesthetic

**The Problem:** LLMs default to dark backgrounds with purple/violet gradients. It's become the single most recognizable "AI-made" aesthetic.

**What it looks like:** Dark (#0a0a0f or similar) background, purple-to-blue gradient accents, glowing text effects, purple tinted cards.

**Screenshot:** See `screenshots/nunu_hero.jpg` — Nunu.ai's homepage uses a dark purple gradient with glowing accent lines. (Video timestamp 3:10)

**The Fix:** Choose a color palette that reflects your actual brand. If you don't have brand colors yet, pick something distinctive — even a simple white/light background with one bold accent color stands out more than the default AI purple.

---

### 2. Decorative SVG Line Animations

**The Problem:** LLMs are good at generating SVG path animations, so they default to adding animated lines that weave through page sections. These add no informational value and actively distract from content.

**What it looks like:** A thin animated line (often purple or gradient) that curves through sections of the page, connecting elements with no semantic meaning.

**Screenshot:** See `screenshots/nunu_line.jpg` — an animated line snakes vertically through the page. (Video timestamp 3:40)

The reviewers noted the line is "distracting" — just because LLMs are good at SVG doesn't mean it helps convert visitors.

**The Fix:** Remove decorative line animations entirely. Use clear headings, whitespace, and logical content ordering instead.

---

### 3. System Emojis as Icons

**The Problem:** LLMs use system emojis as quick icon replacements. This is immediately recognizable as lazy AI output.

**Screenshot:** See `screenshots/rosebud_emojis.jpg` — Rosebud AI's navigation tabs use system emojis, which the reviewer calls "immediately kind of like a tell." (Video timestamp 12:00)

**The Fix:** Use a proper icon library (Lucide, Heroicons, Phosphor) or custom SVG icons. Text-only is cleaner than emoji-decorated.

---

### 4. Excessive Hover Effects

**The Problem:** AI tools add hover effects to everything. This makes the page feel fidgety rather than polished.

**Screenshot:** Discussed at video timestamp 6:30 on Nunu.ai — hover effects on navigation are something "you get for free even without CSS. It turns the cursor into a pointer. That's kind of all you need."

**The Fix:** Hover effects should be subtle and purposeful. For links: a subtle color change or underline. For cards: a slight shadow lift at most.

---

### 5. Scroll Jacking

**The Problem:** AI tools override the browser's native scroll behavior. Users hate this because it takes away their control.

**Screenshot:** See `screenshots/zarna_hero.jpg` — Zarna (timestamp 30:35) had scroll jacking that feels like "moving through molasses." Also seen on `screenshots/getcrux_hero.jpg` (timestamp 13:50).

**The Fix:** Never override native scroll behavior. Smooth-scroll CSS on anchor navigation is fine; stealing the user's scroll input is not.

---

### 6. Fade-In Animations on Every Section

**The Problem:** LLMs apply fade-in-on-scroll to every section. Animating everything signals "the AI did this because it could."

**The Fix:** Use entrance animations sparingly — maybe on the hero section or one key CTA. The rest should be immediately visible.

---

### 7. Fake Dashboard Screenshots with "Google Colors"

**The Problem:** AI tools generate fake dashboards using red/green/blue/yellow (the "Google colors"). These are a hallmark tell.

**Screenshot:** See `screenshots/build0_dashboard.jpg` — Build0's dashboard shows a "Bug & QA Tracker" with this exact color pattern. (Video timestamp 27:50)

The reviewer noted: "Every fake dashboard looks basically like that... it's usually like the Google colors."

**The Fix:** Use actual product screenshots. If the product isn't built yet, create a carefully designed mockup with YOUR color palette.

---

### 8. Bento Box Grid Layouts

**The Problem:** The bento-box layout is not bad, but LLMs default to it constantly, making it feel generic.

**Screenshot:** See `screenshots/build0_dashboard.jpg` — Build0 uses bento grid for features. (Video timestamp 28:00)

The reviewer noted bento boxes are "not a bad pattern, but it's also very non-original."

**The Fix:** Consider alternative layouts: vertical feature stacks with screenshots, comparison tables, or interactive demos.

---

### 9. Too Many Typography Styles / Poor Hierarchy

**The Problem:** LLMs introduce extra text styles that a human designer would never add, creating visual noise.

**Screenshot:** See `screenshots/sphinx_hierarchy.jpg` — Sphinx's hero has 5 different text styles. (Video timestamp 20:10)

The reviewer noted: "The LLM thought it's clever to have like an extra label here in an extra style... this just adds a fifth."

**The Fix:** Three to four type styles maximum in any section. One H1, H2 for sections, body text, and maybe one accent style for CTAs.

---

### 10. Weak or Missing Value Proposition in the H1

**The Problem:** The hero H1 should answer: What is it? Who is it for? Why should they care?

The reviewer noted: "What's important for the H1 is typically: what is it, who is it for, and to what end."

**The Fix:** Write your H1 to pass the "stranger test" — would someone with zero context understand what your product does?

## Quick Checklist: Does Your Site Look Vibe-Coded?

- [ ] Dark background with purple/violet gradient?
- [ ] Decorative animated SVG lines?
- [ ] System emojis used as icons?
- [ ] Hover effects on everything (especially nav items)?
- [ ] Scroll jacking or overridden scroll behavior?
- [ ] Fade-in animation on every section?
- [ ] Fake dashboard with red/green/blue/purple color cards?
- [ ] Bento box grid layout for features?
- [ ] More than 3-4 typography styles in a single section?
- [ ] Generic H1 that doesn't answer "what, who, why"?
- [ ] "Google colors" in any data visualization?

If you check 3+ of these, the site needs a design pass to remove vibe-coded patterns.

## Guiding Principle

**Just because an AI tool can add something doesn't mean you should keep it.** Every element should earn its place. The best AI-assisted websites are the ones where you critically review the AI's output, remove what's gratuitous, and add only what serves the user's understanding and trust. Design is as much about what you take away as what you add.
